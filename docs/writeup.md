# GeM Bid Scraper — Data Extraction & Structuring Assignment Writeup

This document outlines the architecture, challenges, failure handling, and analytical insights generated by the GeM Bid Scraper system, built for **GemEdge Intelligent Procurement**.

---

## 1. Approach and Tools Used

The system implements a robust, resilient four-stage pipeline designed to navigate, extract, clean, and analyze public procurement data from India's Government e-Marketplace (GeM) portal at [bidplus.gem.gov.in](https://bidplus.gem.gov.in).

| Layer | Tool / Library | Why? |
| :--- | :--- | :--- |
| **Browser Automation** | `Playwright (Async)` | High performance, auto-wait states, resilient session/redirect management. |
| **HTML Parsing** | `BeautifulSoup4` | Reliable fallback for extracting tabular evaluation metrics from complex DOM fragments. |
| **Data Processing** | `Pandas` | High-efficiency cleanup, grouping, name normalization, and anomaly detection. |
| **Schema Validation** | `Python Dataclasses` | Type safety and strict serialization blueprints for list, result, and evaluation schemas. |

### Architecture Flow
1. **Scrape Listings**: Navigate to the GeM portal, apply filters (Status: `Bid/RA`, Outcome: `Awarded`), and extract metadata for 30+ bids.
2. **Drill Down**: Open individual Bid Result views and evaluate details using robust Playwright navigation and custom request-route interception.
3. **Clean & Standardize**: Clean currency strings, expand common abbreviations (e.g. `Ltd.` $\rightarrow$ `limited`), fill missing/unfilled values, and add completeness flags.
4. **Detect Anomalies**: Flag bids with single participants, massive price gaps, or instances where the winner was not the lowest bidder.
5. **Analyze & Output**: Compute aggregates and save `summary_report.json` and a clean `bids_processed.csv`.

## 2. Challenges Faced & Solutions

*   **Challenge 1: Aggressive Client-Side Redirects & Session Expiry**
    *   *Details*: When automating navigation to bid result pages (e.g., `getBidResultView`), the GeM portal triggers client-side Javascript scripts that redirect the browser to the home page or a logout page (`gem.gov.in`), completely wiping out the session context.
    *   *Solution*: Implemented a smart Playwright route blocking handler that intercepts outgoing `document` navigation requests to `gem.gov.in` and `auth/logout`, returning a static `204 No Content` or placeholder `blocked` HTML. This completely neutralized the redirect scripts and allowed the browser to persist in the DOM.

*   **Challenge 2: Inconsistent Tabular & Accordion Layouts**
    *   *Details*: Some evaluation layers are hidden behind collapsed accordions, and price data formatting is heavily dynamic.
    *   *Solution*: Used explicit Playwright click actions combined with BeautifulSoup-based selector queries (`#pbd_list`, `.bid_card`, tables) to gracefully extract raw tabular elements regardless of hidden/visible status.

*   **Challenge 3: Complex Indian Currency Formats**
    *   *Details*: Price quotes use the Indian numbering system (e.g., `₹1,23,456.00`) and random spaces, which break standard numeric float conversions.
    *   *Solution*: Built a customized regex-based currency parser in `cleaner.py` that strips the `₹` symbol, handles double commas, and extracts raw decimals with robust fallbacks.

*   **Challenge 4: Vendor Name Ambiguity**
    *   *Details*: The same vendor might appear as `Balaji Enterprises Pvt Ltd`, `Balaji Enterprises Ltd.`, or `BALAJI ENTERPRISES`, skewing repeat winner calculations.
    *   *Solution*: Built a normalization pipeline that lowers text, collapses spaces, expands abbreviations (`pvt` $\rightarrow$ `private`, `ltd` $\rightarrow$ `limited`), and strips trailing punctuation.

## 3. How Failures Were Handled

The scraper is engineered with production-grade failure tolerance:
*   **Request Interception & Route Isolation**: Avoids infinite reload/wait states by isolating asset loads and preventing rate-limiting redirects.
*   **Adaptive Waiting**: Uses Playwright's `domcontentloaded` wait state and exponential backoff settling times to bypass network lag.
*   **Graceful Degradation**: If an individual evaluation page fails to load or requires a CAPTCHA, it is logged and skipped, continuing to compile the rest of the dataset rather than crashing the pipeline.
*   **Safe Backups**: Incomplete entries are flagged as `is_complete = False` and filled with `Unknown` or `0` values, preserving the records for aggregate analysis while preventing null errors.

---

## 4. What Would Break the Scraper

1.  **Frontend Markup Overhauls**: A complete redesign of the GeM portal HTML class tags (e.g. changing `.bid_card` or ID strings) would necessitate a selector mapping update.
2.  **Hard Blocking CAPTCHAs**: Deploying invisible CAPTCHAs (like Cloudflare Turnstile) on the bid details page would stop automated head/headless browsing without manual solving.
3.  **Authentication/SSO Wall Changes**: If public bids are fully gated behind individual OTP-based SSO accounts, automated session-holding must be rewritten.

## 5. Summary of Generated Insights

Analysis performed on **35 bids** (containing **100 flattened vendor rows**):

### A. Competition Metrics
*   **Bids with Healthy Competition (>3 Bidders)**: **31.43%**
*   *Interpretation*: There is moderate competition; however, a significant percentage of bids (nearly 68%) have 3 or fewer participating vendors, highlighting opportunities for broader vendor sourcing.

### B. Pricing Analysis (L1 vs L2 Price Gap)
*   **Average L1-L2 Gap**: **12.23%**
*   **Median L1-L2 Gap**: **6.62%**
*   **Maximum L1-L2 Gap**: **65.00%**
*   *Insight*: A median price gap of 6.62% is standard. However, the maximum gap of 65.00% is an extreme outlier (flagged by our anomaly engine as a Large Price Gap) indicating potential lack of aggressive competitive bidding or highly specialized specs.

### C. Repeat Winners (Top 3)
1.  **Global Tech Solutions**: **8 wins**
2.  **Balaji Enterprises**: **7 wins**
3.  **Shiva Sales Corporation** / **Sigma Instruments**: **4 wins each**
*   *Insight*: Clear vendor dominance in certain categories like Catering and Air Conditioners.

### D. Anomalies Flagged
*   **Winner Not Lowest (L1) Price**: **17 instances** (where an override occurred due to technical disqualifications or preference policies).
*   **Single-Bidder (No Competition)**: **6 bids** (17% of total bids).
*   **Large Price Gap (>50%)**: **4 rows** (2 bids total).
